MySQL 핵심 개념 정리

        
문제를 풀기 전에 자주 헷갈리는 문법을 여기서 따로 확인하세요.

      

      
    


    

      

        
1. 테이블명.컬럼명 개념

        
Book.bookid, Orders.bookid처럼 쓰는 것은 “어느 테이블의 컬럼인지” 정확히 지정하는 문법입니다.

        
SELECT Book.bookname, Orders.orderid
FROM Book JOIN Orders
ON Book.bookid = Orders.bookid;

        
두 테이블에 같은 이름의 컬럼이 있을 때는 반드시 구분해주는 것이 좋습니다.

      


      

        
2. JOIN 기본 개념

        
JOIN은 여러 테이블을 연결해서 한 번에 조회하는 문법입니다. 연결 기준은 보통 ON에 씁니다.

        
FROM Customer JOIN Orders
ON Customer.custid = Orders.custid

        
custid가 같은 행끼리 붙어서 고객 이름과 주문 정보를 함께 볼 수 있습니다.

      


      

        
3. LEFT OUTER JOIN

        
왼쪽 테이블은 전부 살리고, 오른쪽에 맞는 데이터가 없으면 오른쪽 컬럼은 NULL로 나옵니다.

        
SELECT Book.bookname, Orders.orderid
FROM Book LEFT JOIN Orders
ON Book.bookid = Orders.bookid;

        
예: 주문된 적 없는 도서도 조회하고 싶을 때 사용합니다.

      


      

        
4. RIGHT OUTER JOIN

        
오른쪽 테이블은 전부 살리고, 왼쪽에 맞는 데이터가 없으면 왼쪽 컬럼은 NULL로 나옵니다.

        
SELECT Book.bookname, Orders.orderid
FROM Book RIGHT JOIN Orders
ON Book.bookid = Orders.bookid;

      


      

        
5. MIN / MAX

        
MIN은 최솟값, MAX는 최댓값을 구하는 집계 함수입니다.

        
SELECT MIN(price) FROM Book;
SELECT MAX(price) FROM Book;

        
GROUP BY와 함께 쓰면 그룹별 최솟값/최댓값도 구할 수 있습니다.

        
SELECT custid, MAX(saleprice)
FROM Orders
GROUP BY custid;

      


      

        
6. WHERE vs HAVING

        
WHERE는 그룹으로 묶기 전 개별 행 조건, HAVING은 GROUP BY 이후 집계 결과 조건입니다.

        
SELECT custid, COUNT(*)
FROM Orders
WHERE saleprice >= 10000
GROUP BY custid
HAVING COUNT(*) >= 2;

      


      

        
7. NULL 비교

        
NULL은 =, !=로 비교하지 않고 IS NULL, IS NOT NULL을 사용합니다.

        
WHERE phone IS NULL
WHERE phone IS NOT NULL

      


      

        
8. LIKE 와일드카드

        
_는 정확히 한 글자, %는 글자 수 제한 없음을 의미합니다.

        
WHERE name LIKE '박%'
WHERE bookname LIKE '%축구%'
